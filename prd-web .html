1 产品概述

1.1 产品定位与愿景

本产品定位为家庭智能陪伴机器人解决方案，通过将大语言模型与宇树G1人形机器人深度融合，打造具有情感交互能力、动作响应能力、任务执行能力的智能陪伴助手。产品核心愿景是实现"居家保姆机器人"的终极目标，将养老服务、儿童托管、家庭日常事务管理等功能集于一身，成为真正的家庭智能伙伴。

核心产品定位

"具身智能+" — 让每个家庭都能拥有一个不知疲倦、专业可靠、全天候待命的智能保姆

3.3 用户身份认证系统 V2.0

双重认证机制

本系统采用声纹比对+人脸识别双重认证机制，确保身份识别的准确性和安全性。声纹比对作为第一道防线，人脸识别作为第二道防线，双重验证均通过后用户身份确认完成。

graph LR subgraph 双重认证流程 A1[用户进入] --> A2[声纹比对] A2 --> A3{声纹置信度达标?} A3 -->|是| A4[人脸二次验证] A3 -->|否| A5[重新声纹采集] A4 --> A6{人脸验证通过?} A6 -->|是| A7[身份确认成功] A6 -->|否| A8[重新验证] A7 --> A9[加载用户配置] end style A2 fill:#e0e7ff style A4 fill:#c7d2fe style A7 fill:#a7f3d0
方案一：机器人端语音引导注册
  1. 1. 机器人开机播放欢迎语引导注册
  2. 2. 用户朗读几句话（声纹采集与注册）
  3. 3. 用户面对摄像头（人脸采集与注册）
  4. 4. 语音输入姓名和身份角色
  5. 5. 注册完成，自动开始服务

适用场景：所有用户，尤其老年人

方案二：提前实名认证（推荐）
  1. 1. 手机扫码进入实名认证页面
  2. 2. 上传身份证+人脸活体检测
  3. 3. 输入姓名、选择身份角色
  4. 4. 生成设备绑定码
  5. 5. 下载APP后开机即用，无需再次注册

适用场景：追求效率的用户，开箱即用

声纹比对认证特点
  • • 非接触式采集，用户无感知
  • • 难以伪造复制，安全性高
  • • 持续验证，对话中自动确认
  • • 响应速度快（<500ms）
人脸识别认证特点
  • • 高精度核验，双重保障安全
  • • 支持活体检测，防照片欺骗
  • • 环境光自适应调整
  • • 单次比对延迟<1秒
动态用户识别
  • • 持续声纹比对监测
  • • 身份变化自动切换
  • • 个性化服务配置
  • • 多用户权限管理
双重认证性能指标
  • • 声纹比对延迟：<500ms
  • • 人脸比对延迟：<1s
  • • 联合验证总延迟：<1.2s
  • • 声纹置信度阈值：0.85

4 用户交互路径

4.1 典型使用场景

场景1

老年人日常陪伴与健康提醒

用户走进客厅 → 声纹识别+人脸验证确认身份 → 切换老年人模式 → 用户说"提醒我下午3点吃药" → 机器人确认并执行点头动作 → 执行前15分钟APP提醒 → 执行时间到达 → 机器人走到身边提醒"爷爷，吃药时间到了"

场景2

儿童睡前故事

家长APP设置"睡前故事"模式 → 识别儿童用户 → 切换儿童友好模式 → 灯光调暗 → 机器人讲故事配合动作 → 小朋友中途打断提问 → 立即响应并继续讲述

场景3

家庭日常事务管理

用户说"帮我安排明天的日程" → 机器人调出日程管理 → 语音播报安排 → 用户创建提醒任务 → APP同步接收 → 按时执行各项提醒

场景4

家庭安全监控

检测到陌生人 → 人脸识别确认非家庭成员 → APP告警通知 → 家庭成员查看访客图像 → 选择"确认进门"或"拒绝访问" → 机器人执行相应指令

核心交互设计原则

• 自然优先：开口即用，无需学习
• 容错宽容：询问确认，引导用户
• 隐私安全：加密传输，用户可控
• 无障碍：适老适幼，降低门槛

7 非功能性需求

性能指标

  • VAD检测延迟<100ms
  • 端到端总延迟<1.5s
  • 声纹比对延迟<500ms
  • 人脸比对延迟<1s
  • 双重认证总延迟<1.2s
  • 系统吞吐量≥50并发

安全性要求

  • •TLS加密传输，WSS协议
  • •声纹+人脸多因素双重认证
  • •指令合法性审核机制
  • •生物特征数据加密存储
  • •支持数据导出和删除

可用性要求

  • •系统可用性：99.9%
  • •多可用区容灾部署
  • •本地降级能力
  • •7x24小时监控告警

兼容性要求

  • •宇树G1全系列机型
  • •宇树ROS环境和SDK
  • •主流推理框架和模型
