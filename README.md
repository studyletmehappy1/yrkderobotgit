## 一、产品概述

### 1.1 产品定位与愿景

本产品定位为**家庭智能陪伴机器人解决方案**，通过将大语言模型与宇树G1人形机器人深度融合，打造具有情感交互能力、动作响应能力、任务执行能力的智能陪伴助手。产品核心愿景是实现**"居家保姆机器人"**的终极目标，将养老服务、儿童托管、家庭日常事务管理等功能集于一身，成为真正的家庭智能伙伴。

区别于传统语音助手和简单的陪伴机器人，本产品强调**"具身智能+"**概念，不仅具备AI的认知能力，还拥有机器人的物理执行能力，能够理解用户情感、执行家庭任务、照顾老人儿童、保持持续对话互动。产品目标是让每个家庭都能拥有一个不知疲倦、专业可靠、全天候待命的智能保姆，让科技真正服务于日常生活。

### 1.2 核心价值主张

本产品的核心价值主张包含六个维度。第一是**自然交互体验**，通过近场语音采集、实时语音识别、情感语音合成等技术，实现人与机器人之间如同人与人交流般的自然对话体验。第二是**智能安全过滤**，基于大语言模型的语义理解能力，系统能够准确区分用户的真实指令与环境噪音，过滤无效输入，同时进行指令合法性判断，确保机器人不执行任何违法或危险指令。第三是**身份动态感知**，通过人脸识别和声纹识别的双重认证机制，系统能够动态识别当前对话用户，提供个性化服务并保障家庭安全。第四是**全自动化任务**，支持定时任务设置和执行，用户可通过语音下达定时提醒、家务安排等任务，APP端提供智能提醒和执行确认。第五是**动作响应能力**，机器人能够通过预设动作库和自定义动作编排，在对话过程中配合相应的肢体动作，增强交互的真实感和感染力。第六是**灵活部署架构**，通过宇树unistore生态，实现一键部署、即插即用的便捷体验。

### 1.3 目标用户分析

本产品的目标用户分为三大群体，每个群体都有其独特的需求和期望。

**独居老人群体**是产品的核心服务对象之一。这类用户通常对智能设备操作不熟悉，但又存在强烈的陪伴需求和安全保障需求。智能陪伴机器人可以提供日常对话陪伴、情感关怀支持、健康提醒服务、用药提醒设置、紧急呼叫功能等。同时子女可以通过手机APP远程查看机器人状态、与老人进行视频通话、设置老人的日程提醒等。机器人的语音交互设计充分考虑了老年人的特点，包括大音量输出、慢速语音合成、方言支持等。

**儿童教育陪伴群体**是产品的另一核心用户。家长希望孩子在成长过程中有一个能够互动学习、娱乐陪伴的伙伴。机器人可以通过讲故事、知识问答、互动游戏、作业辅导等方式陪伴儿童，同时家长可以设置学习内容和时间限制，监控儿童的观看内容。机器人还会根据儿童的年龄和兴趣，提供适龄的教育内容，并通过动作互动增强学习的趣味性。

**智能家居控制群体**将机器人作为智能家居的控制中枢和家庭事务管家。通过语音指令控制家中的智能设备，实现动口不动手的便捷生活方式。同时机器人可以承担家务提醒、日程管理、购物清单维护等家庭事务管理工作，真正成为家庭的好帮手。

## 二、系统架构设计

### 2.1 整体技术架构

本系统采用**前后端分离、云端协同、端云一体**的复合技术架构设计方案。在V2.0版本中，系统架构进行了全面升级，增加了用户身份认证层、指令安全过滤层、定时任务管理层等核心组件。整体架构分为五个层次：感知层、处理层、服务层、执行层和展现层。

**感知层**部署在机器人本地，承担环境感知和用户输入采集任务。包括麦克风阵列采集语音输入、摄像头采集人脸图像、传感器采集环境状态等。感知层的核心设计理念是"本地预处理、云端深度分析"，基础的信号处理在本地完成，更复杂的语义理解在云端进行。

**处理层**包含语音识别（ASR）、语音活动检测（VAD）、声纹识别、图像处理等核心算法模块。VAD模块负责实时检测用户是否在说话，为后续的ASR识别提供准确的语音片段输入。声纹识别模块提取用户声音特征，用于动态识别当前对话用户身份。

**服务层**是系统的核心智能中枢，包含大语言模型推理服务、用户账户管理服务、定时任务调度服务、指令安全过滤服务等。大语言模型负责理解用户意图、生成智能回复、协调动作执行。账户管理服务维护家庭用户档案，支持人脸和声纹的多因素认证。定时任务服务管理和执行各类定时提醒和周期性任务。

**执行层**负责将服务层的决策转化为具体的物理动作。包括动作指令执行器（控制机器人肢体动作）、音频播放模块（输出语音反馈）、IoT控制模块（操控智能家居设备）、APP通知模块（发送移动端提醒）等。

**展现层**为用户提供多渠道的交互界面。包括机器人本体的语音和动作交互、手机APP的图形界面交互、Web管理后台的配置界面等。展现层的设计理念是"无感切换"，用户可以在不同渠道之间无缝切换使用。

### 2.2 模块职责划分

**客户端本地模块**承担实时性要求高和隐私敏感的任务处理。语音采集模块负责从麦克风阵列获取高质量音频数据，配置有回声消除、噪声抑制等功能。语音识别模块（ASR）基于本地或云端API完成语音转文字任务，支持多语言和多方言。语音活动检测模块（VAD）采用深度学习算法准确区分人声与环境噪音。声纹识别模块提取用户声音特征用于身份验证。人脸识别模块通过摄像头采集图像并进行身份核验。指令过滤引擎对识别文本进行初步的意图判断和噪音过滤。音频播放模块负责接收云端合成的语音数据并通过扬声器输出。动作指令执行器负责解析云端返回的动作指令并通过上位机协议发送给机器人运动控制系统。

**云端服务端模块**负责计算密集型的AI推理任务和业务逻辑处理。API网关是系统的统一入口，承担认证鉴权、流量控制、负载均衡等职责。大语言模型推理服务是系统的核心，基于业界领先的大模型理解用户意图、生成自然流畅的回复。指令安全过滤模块对每条用户指令进行合法性检查，确保系统不执行任何违法或危险指令。用户账户管理服务维护家庭用户档案、权限设置、个性化配置等信息。定时任务调度服务管理和执行各类定时任务，支持复杂的调度规则。回复生成引擎在接收到大模型的输出后，进行后处理操作生成包含文本、动作、提醒等信息的综合响应。语音合成服务将文本回复转换为自然流畅的语音输出。动作指令解析模块从文本回复中提取动作指令，生成结构化的动作序列。

### 2.3 数据流向设计

系统的数据流向遵循"身份验证→语音输入→本地处理→云端推理→指令审核→动作执行→语音输出→任务提醒"的完整闭环。整体端到端延迟目标控制在2秒以内，确保用户感受到的是流畅自然的对话体验。

用户说话后，系统首先进行身份识别（声纹+人脸双重验证），确认用户身份后进行语音信号处理。VAD模块实时分析音频流，检测语音的起止位置。ASR模块将语音转换为文本后，首先经过本地的初步过滤引擎进行噪音识别和意图判断。通过初步过滤的有效指令被发送到云端API网关，网关进行安全校验后分发给大模型推理服务。大模型生成的回复包含文本内容、动作指令、情感标签、定时任务等信息。回复生成引擎对这些信息进行解析和重组，指令安全过滤模块对动作指令进行二次审核。审核通过后，语音合成服务生成音频数据，动作指令发送到动作执行队列。APP端同时接收任务提醒数据，在定时任务执行前15分钟发送通知。最终，音频数据通过扬声器播放，动作指令通过机器人控制接口执行。

## 三、核心功能规格

### 3.1 语音交互系统

语音交互系统是本产品的核心功能模块，负责实现用户与机器人之间的语音沟通。V2.0版本在原有基础上增加了多用户识别支持和指令安全过滤功能。

**语音活动检测模块（VAD）**负责实时分析音频流，检测用户是否正在说话。VAD模块采用深度学习算法，支持多种音频场景的自适应切换。在家庭环境中，VAD需要处理各种复杂的声学场景，包括多人同时说话的鸡尾酒会场景、背景音乐播放场景、家庭成员之间的交谈等。VAD模块的性能指标要求是语音检测准确率达到95%以上，语音起始延迟小于100毫秒，语音结束判断延迟小于300毫秒。

**语音识别模块（ASR）**负责将用户的语音信号转换为文本内容。系统支持多种ASR服务提供商的API调用，用户可根据实际需求选择适合的服务商。

**语音合成模块（TTS）**负责将机器人的文本回复转换为自然流畅的语音输出。系统支持多种TTS服务提供商的API调用，提供多种音色选择和情感表达功能。

### 3.3 智能指令过滤系统

智能指令过滤系统是本产品的核心技术模块，它解决的核心问题是"如何区分用户指令和环境噪音"以及"如何确保指令的合法性和安全性"。

**多层级噪音过滤策略**。第一层是基于声学特征的过滤，系统通过分析音频的频谱特征、能量分布、音色特点等声学参数，判断输入是否具有人声特征。第二层是基于唤醒词和唤醒声的过滤，用户必须首先说出唤醒词或做出特定的唤醒动作，系统才会进入指令接收状态。第三层是基于语义理解的过滤，即使语音识别模块成功识别出一段文本，系统仍然会通过语义分类模型判断这段文本是否构成有效指令。第四层是基于上下文的过滤，系统会结合对话历史和当前场景上下文，判断当前输入是否符合对话逻辑。

**指令合法性判断机制**是V2.0版本新增的核心安全功能。系统内置指令安全过滤模块，对每一条用户指令进行合法性审核。审核维度包括：指令内容是否涉及违法行为的请求（如伤害他人、盗窃、诈骗等）；指令是否可能导致机器人自我伤害或损坏（如要求机器人撞击物体、执行危险动作等）；指令是否涉及敏感信息的索取或泄露；指令是否违反家庭安全规则（如要求打开危险物品等）。

当系统检测到不合法的指令时，会采取分级响应策略。对于轻度违规指令（如要求机器人做出不当动作），机器人会以温和的方式拒绝并解释原因；对于严重违规指令（如明确要求实施违法行为），机器人会立即拒绝并可能触发安全告警。系统会记录所有违规指令的请求日志，供管理员审查。

### 3.4 用户身份认证与动态识别系统

V2.0版本新增用户身份认证系统，通过人脸识别和声纹识别的双重认证机制，实现用户"开箱即注册"和动态身份识别功能。

**开箱即注册机制**是用户体验的重要创新。当新用户首次与机器人交互时，机器人会通过语音引导用户完成简单的注册流程：首先是声纹注册，用户对着机器人说几句话，系统提取声纹特征并保存；然后是人脸注册，摄像头采集用户面部图像，系统提取人脸特征并保存。整个过程通过语音引导完成，无需用户操作任何设备或界面。注册完成后，系统自动为用户分配默认的家庭成员身份（可根据需要调整），用户档案包含声纹特征、人脸特征、姓名、身份角色、个性化偏好设置等信息。

**动态用户识别**功能让机器人能够识别当前对话的用户身份。在对话过程中，系统持续进行声纹比对，动态确认当前说话者的身份。当检测到用户身份发生变化时（如家庭成员替换），系统会自动更新对话上下文，并可以根据新用户的身份调整服务内容和交互风格。例如，如果识别到是家中的儿童用户，机器人会自动切换到更适合儿童的对话模式和内容。

**多用户管理**支持家庭中多个成员分别注册和识别。系统维护每个家庭成员的用户档案，支持为不同成员设置不同的权限级别和安全规则。例如，可以设置儿童用户无法访问某些敏感功能，或者老年人用户自动开启大音量模式等。

### 3.5 连续打断机制

连续打断机制是语音交互系统的重要功能，解决的核心问题是"如何在机器人说话时允许用户随时打断"。在自然对话中，打断是一种常见的沟通方式，本产品设计了完整的连续打断机制，实现"想说什么就说什么"的无障碍沟通体验。

**打断检测**采用唤醒词优先策略，用户可以通过说出唤醒词来触发打断。同时，系统还支持特定的打断关键词，如"停"、"等等"、"不对"等，这些词汇被设计为优先级最高的打断信号。当检测到打断信号时，系统立即执行中断操作。中断执行需要在100毫秒内完成语音停止，清空待发送的音频队列。

**打断后的状态恢复**采用"静默等待"策略，打断发生后，机器人保持安静状态，等待用户的下一条指令。如果用户在打断后没有立即说话，系统会在3秒后发出简短的提示音。

### 3.6 动作响应系统

动作响应系统赋予机器人通过肢体动作表达的能力，动作响应系统与语音交互系统紧密配合，增强机器人的表现力和亲和力。

**动作库管理**模块负责管理和维护机器人可执行的动作集合。宇树G1机器人的动作库包含预定义动作和自定义动作两类。预定义动作包括：打招呼动作（挥手、点头、鞠躬等）、情绪表达动作（开心时的手舞足蹈、困惑时的挠头等）、日常动作（站立、坐下、转向、伸手等）、功能性动作（拾取物品、指向方向等）。自定义动作允许用户通过动作编辑器编排新的动作序列。

**动作编排与模型反馈**是V2.0版本的增强功能。在前期实现阶段，动作编排主要基于规则匹配：大模型生成回复文本时，系统分析文本的情感色彩和语义内容，匹配相应的预设动作。例如，当文本内容表达"欢迎"时，匹配"招手"动作；当文本表达"思考"时，匹配"挠头"动作。

在后期实现阶段，系统将根据模型的反馈灵活实现各种动作。大模型的输出将包含结构化的动作指令信息，动作编排模块解析这些指令并生成精确的动作序列。这种设计允许更灵活和自然的动作配合，因为大模型可以根据对话上下文理解何时需要动作、需要什么样的动作。

**动作指令反馈机制**是实现"语音+动作"沟通的关键。系统建立了动作执行反馈闭环：云端生成动作指令后，发送给机器人执行；机器人执行动作并反馈执行状态；执行状态回传给云端用于后续对话决策。这种机制确保了语音和动作的一致性和协调性，避免出现声音和动作不匹配的情况。

### 3.7 定时任务系统

定时任务系统是实现"居家保姆"功能的核心模块，支持用户通过语音下达定时任务，机器人按时执行并反馈。

**定时任务类型**包括以下几类：健康提醒类任务，如"提醒我每天早上8点吃药"、"提醒爷爷下午3点测量血压"等；生活作息类任务，如"提醒奶奶晚上9点睡觉"、"提醒孩子每天7点起床"等；家务管理类任务，如"每天下午5点提醒我做饭"、"每周六上午提醒我打扫卫生"等；自定义提醒类任务，用户可设置任意时间和内容的提醒。

**语音任务下达**采用自然语言交互方式，用户可以直接说"明天早上8点提醒我开会"、"以后每天晚上8点提醒我吃药"等，系统会自动解析时间信息和任务内容，并创建对应的定时任务。系统支持一次性任务和周期性任务两种模式，支持复杂的周期设置（如"每两天"、"每周一三五"等）。

**APP端智能提醒**是定时任务系统的重要组成部分。在具体任务执行前15分钟，APP会自动发送消息提醒，弹出"执行"和"暂停执行"两个选项供用户选择。如果用户在任务执行时间前未做出选择，系统将默认执行该任务（除非用户在设置中修改了默认行为）。

**任务执行反馈**机制确保用户了解任务执行状态。任务执行后，机器人会通过语音向用户确认"您的提醒已执行"或"吃药时间到了"。同时，APP端会收到任务执行的通知，用户可查看任务执行历史记录。

## 四、用户交互路径

### 4.1 典型使用场景

**场景一：老年人日常陪伴与健康提醒**

用户（老年人）走进客厅，机器人通过声纹识别确认用户身份，自动切换到适合老年人的交互模式（慢速语音、大字体显示等）。用户说"小宇小宇，提醒我下午3点吃药"，机器人回应"好的，已设置下午3点提醒您吃药。还需要我做什么吗？"同时执行点头确认动作。下午2:45，系统通过APP向用户子女发送提醒"爷爷的吃药时间将在15分钟后执行，默认将执行该任务"。下午3:00，机器人走到用户身边，播放提醒语音"爷爷，吃药时间到了"，并执行伸手递药的动作示意。

**场景二：儿童睡前故事**

家长通过手机APP设置"睡前故事"模式，并指定由家中的小朋友接收服务。机器人识别到儿童用户后，自动切换到儿童友好模式（活泼语音、有趣动作等），灯光调暗进入舒适氛围。机器人开始讲故事"从前有一只小兔子..."，配合讲故事的语境执行相应的动作，如模仿小兔子蹦跳、做出惊讶的表情等。讲到一半，小朋友打断说"小兔子后来怎么了？"，机器人立即停止当前故事，温柔回应"小兔子后来遇到了一个新朋友..."并继续讲述。如果小朋友忘记时间太晚，家长可以通过APP远程提醒机器人"时间差不多了，该睡觉了"。

**场景三：家庭日常事务管理**

用户说"小宇，帮我安排一下明天的日程"，机器人调出日程管理界面并语音播报"明天是周六，您有以下安排：上午9点有家庭聚会需要准备食材；下午3点需要去接孩子放学；晚上7点和朋友聚餐"。用户继续说"帮我提醒一下，在聚会开始前1小时提醒我准备食材"，系统自动创建提醒任务。APP端同步收到日程信息，用户可在手机上查看和修改。第二天，系统按时执行各项提醒任务。

**场景四：家庭安全监控**

机器人识别到有陌生人进入家中，通过人脸识别确认不是家庭成员。机器人通过APP向家庭成员发送告警通知"检测到访客，请确认身份"。家庭成员可通过APP查看访客图像，选择"确认进门"或"拒绝访问"指令。如果收到"拒绝访问"指令，机器人会礼貌地请访客离开。如果访客是快递员，用户可通过APP远程让机器人代为收取快递。

### 4.2 用户身份认证流程

**新用户开箱注册流程**包括以下步骤：

第一步，机器人开机启动后检测到无注册用户，播放欢迎语"欢迎使用智能陪伴机器人，请完成简单的注册以便我更好地为您服务"。

第二步，声纹注册引导。机器人说"请对我说几句话，这样我可以记住您的声音"。用户朗读简短语句（系统提供标准文本或自由表达），系统提取声纹特征。

第三步，人脸注册引导。机器人说"请面对我的摄像头，让我看看您的样子"。摄像头采集用户面部图像，系统提取人脸特征。

第四步，信息完善。机器人询问"请问您叫什么名字？"，用户回答姓名。系统询问"请问您的身份是？"，用户可选择预设选项（爷爷、奶奶、爸爸、妈妈、孩子、其他）。

第五步，注册完成确认。机器人说"注册完成，很高兴认识您，{姓名}！以后我会记住您的声音和样子"。

**日常使用身份识别流程**：

用户走进机器人感知范围 → 系统自动进行声纹比对（持续进行）→ 识别到已知用户 → 自动切换到该用户的个性化配置 → 如需更高安全级别，启动人脸二次验证 → 确认用户身份后开始服务。

### 4.3 定时任务交互流程

**语音创建任务流程**：

用户说"提醒我..." → 系统进行语音识别 → 本地VAD检测和ASR处理 → 发送文本到云端 → 云端解析时间和任务内容 → 创建定时任务记录 → 确认创建成功 → 用户收到语音反馈。

**APP提醒流程**：

定时任务触发前15分钟 → 系统生成提醒消息 → 推送到用户APP → 用户收到推送通知 → APP显示任务详情和操作选项 → 用户选择执行/暂停/修改 → 操作结果同步到服务端 → 任务执行时间到达 → 机器人执行任务 → 发送执行结果反馈。

## 五、部署架构设计

### 5.1 灵活部署方案

本产品支持多种部署方案，适应不同用户的需求和技术条件。

**基础部署方案**采用纯云端服务，用户只需购买机器人硬件，软件服务全部通过云端API提供，适合技术能力较弱、希望开箱即用的用户。

**进阶部署方案**采用本地推理服务器，大模型运行在本地服务器上，云端只提供模型更新和技术支持服务，适合对数据隐私有较高要求或网络条件不稳定的用户。

**高级部署方案**采用边缘计算，将经过量化的轻量级模型部署到机器人本体上运行，完全脱离网络依赖，适合极端网络环境或对实时性有极高要求的场景。

### 5.2 一键部署机制

通过宇树unistore生态深度整合，实现一键部署的便捷体验。

**unistore部署流程**：用户在unistore平台浏览并选择YRKBot部署包 → 点击"一键部署" → 系统自动完成以下操作：检测目标机器人型号和配置 → 下载并安装所需的软件包 → 配置网络连接和API参数 → 初始化用户认证系统 → 启动本地服务并进行自检 → 显示部署结果和状态信息。

**"机器保姆"理念**：部署包设计遵循"机器保姆一键拉取成功"理念，普通用户无需技术背景，只需几个简单步骤即可完成从购买到使用的全过程。部署包包含完整的客户端应用、本地服务程序、云端连接配置、动作库文件、默认对话模型等所有必要组件。

### 5.3 版本管理与更新机制

系统支持手动更新和自动更新两种方式。手动更新适合技术能力较强的用户，自主选择更新的时机和版本。自动更新会在后台检测新版本，在用户不使用机器人的时段自动下载并安装更新包，更新完成后自动重启服务。

版本管理遵循语义化版本规范，每次更新都有清晰的变更说明。对于重要版本的更新，系统会提前通知用户，并提供回滚机制。

所有客户端请求首先到达API网关，网关完成预处理后再分发给后端服务。网关内置智能路由功能，能够根据请求类型和后端服务负载自动选择最优的服务实例。

### 7.2 可用性要求

系统设计可用性目标为99.9%，即全年非计划停机时间不超过8.76小时。系统采用多级容灾设计：云端服务部署在多个可用区，单个可用区故障不影响整体服务；客户端具备本地降级能力；数据定期备份。

### 7.4 兼容性要求

客户端需要兼容宇树G1机器人全系列机型。软件需要适配宇树机器人ROS环境和SDK接口规范。云端服务需要支持主流的推理框架和模型格式。

|----------|----------|----------|
| 用户期望 | 用户对机器人能力有过高期望 | 清晰说明能力边界，渐进式功能引导 |
| 隐私争议 | 语音交互设备引发隐私担忧 | 隐私优先设计，提供透明隐私控制 |
| 指令安全 | 恶意用户尝试违法指令 | 内置安全过滤，记录违规日志 |

### 9.3 市场风险

| 风险类型 | 具体风险 | 应对策略 |
|----------|----------|----------|
| 竞品压力 | 智能陪伴机器人市场竞争激烈 | 聚焦具身智能差异化，强化语音+动作体验 |
| 成本压力 | 硬件+云服务成本较高 | 提供多种部署方案，优化成本结构 |
